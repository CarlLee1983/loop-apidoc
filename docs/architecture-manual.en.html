cli.py exposes fourteen top-level operations: source handling (preprocess, manifest), URL evidence (catalog-url, select-url, cache-url-pages, cache-url-entry, normalize-html-snapshot, related-url-pages), quality and extraction (assess-sources, verify-extraction), run handling (assemble, validate, score, diff) and the foundry sub-group. File I/O exits: generate/, run/, diff/, preparation/, score/, source_quality/, the URL corpus cache and foundry/ are responsible for writing; the remaining domain logic stays as pure functions wherever possible to keep unit testing simple.
