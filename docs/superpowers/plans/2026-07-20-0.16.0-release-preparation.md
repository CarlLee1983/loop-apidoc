# loop-apidoc 0.16.0 Release Preparation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prepare and commit a verified `loop-apidoc 0.16.0` release candidate for the
claim-level semantic evidence work without pushing or creating a tag.

**Architecture:** Use `scripts/release.py prepare` as the sole version-metadata writer,
then complete release-specific documentation and narrow release tests by hand. Run all
deterministic gates before the release commit, and run the tag dry-run afterward because
the release wrapper requires a clean worktree.

**Tech Stack:** Python 3.11+, uv, pytest/pytest-cov, Ruff, npm, Tagsmith, Markdown, JSON,
TOML.

## Global Constraints

- Target version is exactly `0.16.0`.
- English is primary and Traditional Chinese is the supporting release-note language.
- The default `legacy` architecture remains authoritative; shadow output remains
  observational.
- Source documents remain the only source of truth; no benchmark source may be
  substituted or fabricated.
- Operator-provided benchmark sources are unavailable in this workspace; strict-local
  validation must be disclosed, not reported as passing.
- Total coverage must remain at or above 95%.
- The final release preparation commit contains no product behavior change.
- Do not push and do not create `v0.16.0`.
- Execute inline; do not delegate unless the user explicitly requests subagents.
